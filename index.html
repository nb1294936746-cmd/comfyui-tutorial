🎨 认识 ComfyUI

什么是 ComfyUI？为什么选择它？

ComfyUI 是什么？

ComfyUI 是一款基于 节点式工作流 的 Stable Diffusion 图形用户界面。与传统的 WebUI 不同，它让你用拖拽连线的方式，像拼积木一样构建 AI 图片生成流程。

每个节点执行一个特定功能（加载模型、编码提示词、采样去噪等），通过连线将它们串起来，就形成了完整的生图工作流。

核心优势

  • ✅ 完全可视化 — 清楚看到图片生成的每一步
  • ✅ 灵活可定制 — 想怎么连就怎么连
  • ✅ 资源占用低 — 比 WebUI 更省显存
  • ✅ 工作流可分享 — 一键导入别人的工作流
  • ✅ 更新快 — 最新模型第一时间支持
  • ✅ 完全开源免费 — MIT 许可证
ComfyUI
  • 节点式可视化界面
  • 显存占用更低（约 3-4GB 起步）
  • 工作流可保存、分享、复用
  • 灵活度极高，支持复杂管线
  • 学习曲线稍陡（但值得）
Stable Diffusion WebUI
  • 传统表单界面，上手更快
  • 显存占用较高（约 6-8GB）
  • 参数设置相对固定
  • 插件生态丰富
  • 灵活度有限，复杂管线困难

方式 A：通过启动器

⬆️ 如何更新 ComfyUI？ ▼

在启动器左侧菜单点击「更新」，可以一键更新 ComfyUI 本体和已安装的插件。更新前建议先关闭 ComfyUI。更新后重新启动即可。

⚙️ 显存不足怎么设置？ ▼

在启动配置的「显存优化模式」下拉框中选择合适的选项：

选项对应参数适用显存说明
正常模式--normalvram≥8GB默认模式，速度最快
低显存模式--lowvram4-6GB分段加载模型，较慢但能跑
极低显存--novram2-4GB几乎全靠内存，很慢
仅 CPU--cpu无 GPU纯 CPU 运行，非常慢
💡
不确定选哪个？先用默认的"正常模式"启动试试。如果生图时报 Out of Memory 错误，再改成"低显存模式"。

🧩 核心节点详解

每个节点都有「还原图」+「批注」，像看真实界面一样学习

● Load Checkpoint — 加载模型（工作流起点）

❶ 标题栏

橙红色 = 模型加载类节点。颜色帮你快速识别节点类型。

❷ ckpt_name 下拉框

点击展开选择你的模型文件。模型放在 models/checkpoints/ 目录才会出现在这里。

❸ 三个输出端口

MODEL → 送入 KSampler（核心去噪模型）
CLIP → 送入 CLIP Text Encode（理解提示词）
VAE → 送入 VAE Decode（潜空间↔像素转换）

💡
一个 Checkpoint 就包含了出图所需的全部三个组件，这就是为什么它是工作流的起点。

● CLIP Text Encode — 提示词编码器（正向 & 反向）

❶ 输入端口（蓝色圆点 = 左侧）

clip — 从 Load Checkpoint 的 CLIP 输出端连过来。这条线代表"文本理解能力"。

❷ 文本输入区

直接在框内写英文提示词。技巧：
· (keyword:1.3) — 增强某个词的权重
· (keyword:0.7) — 降低权重
· 逗号分隔多个描述词

❸ 输出: CONDITIONING

编码后的条件向量。
· 正向 → 连到 KSampler 的 positive 输入
· 反向 → 连到 KSampler 的 negative 输入

● KSampler — 采样器（核心中的核心）

❶ 四个输入端口（全部必须连线）

model ← Load Checkpoint 的 MODEL
positive ← 正向 CLIP Text Encode
negative ← 反向 CLIP Text Encode
latent_image ← Empty Latent Image

❷ 核心参数三件套

seed — 随机种子。固定=结果可复现，随机=每次不同
steps — 采样步数。20 步平衡速度质量，30 步更精细
cfg — 引导强度。越大越像提示词但可能过饱和（推荐 5-10）

❸ 采样算法选择

推荐组合：dpmpp_2m + karras
追求最高质量：dpmpp_3m_sde + karras
追求速度：euler_ancestral + normal

❹ denoise 去噪强度（关键！）

1.0 = 从纯噪声生成（文生图用）
0.5 = 保留一半原图（图生图用）
0.3 = 轻微修改（微调用）

❺ 输出：LATENT

去噪完成的潜空间图像 → 送入 VAE Decode 解码为真实图片。

● Empty Latent Image — 空白画布（设置图片尺寸）

❶ 宽×高 — 决定出图尺寸

必须匹配模型，否则出怪图！
· SD 1.5 模型 → 512×512 / 512×768
· SDXL 模型 → 1024×1024 / 1344×768
· 值必须是 8 的倍数

❷ batch_size — 一次生几张

设为 1 最省显存。设为 4 一次出 4 张但显存×4。新手建议从 1 开始。

输出 → KSampler 的 latent_image

输出一张"纯噪声画布"作为 KSampler 的起点。文生图(txt2img)的标配。

● VAE Decode — 潜空间解码（让你看到真实图片）

❶ 两个输入

samples ← KSampler 输出的 LATENT（数学噪声数据）
vae ← Load Checkpoint 的 VAE（解码器）

❷ 输出: IMAGE — 你能看到的真实图片

把潜空间的 4 通道 64×64 数据解码为 3 通道 512×512 的 RGB 图片。这就是从"数学"变成"图片"的魔法步骤。

🔁
反向操作是 VAE Encode（图片→潜空间），用于图生图(img2img)。

● Save Image — 保存图片（工作流终点）

❶ 输入: images

从 VAE Decode 的 IMAGE 输出连过来。这是最终的 RGB 像素图片。

❷ filename_prefix

文件名前缀，保存为 ComfyUI_00001_.png 格式。
保存到 ComfyUI/output/ 目录。

⭐
隐藏功能：保存的 PNG 文件元数据里包含完整工作流！以后把这张 PNG 拖回 ComfyUI 画布，就能恢复当时的全部节点和参数。
Preview Image 节点不保存文件，仅预览。调试时用它更方便。

这是每个工作流的起点。它负责加载你的 Stable Diffusion 模型文件（.safetensors / .ckpt），并一次性输出模型的三个核心组件。

● Load LoRA — 加载 LoRA 微调模型 ▼

LoRA（Low-Rank Adaptation）是一种轻量级微调技术，可以给基础模型添加特定角色、风格或概念，文件仅几十 MB。

📥 输入

参数说明

📤 输出

端口说明

CLIP融合了 LoRA 的 CLIP

● 局部重绘 (Inpainting) — 只改你想改的区域 ▼

❓ 常见问题 FAQ

新手最常遇到的问题和解决方案

启动后浏览器打不开 / 页面空白？ ▼
  1. 检查命令行是否显示 "To see the GUI go to: http://127.0.0.1:8188"

  4. 尝试换浏览器（推荐 Chrome/Edge）
显存不足 (Out of Memory) 怎么办？ ▼
  • · 减小图片尺寸（如 512×512）
  • · 减少 batch_size 为 1
  • · 使用 --lowvram 或 --novram 参数启动
  • · 关闭其他占用显存的程序
  • · 使用 FP16 精度的模型（文件名含 fp16）
  • · 考虑使用 SD 1.5 模型代替 SDXL
# 低显存启动方式
python main.py --lowvram
python main.py --novram # 极端情况
节点显示红色 / 找不到某个节点？ ▼
  • · 红色节点表示缺少对应的自定义节点插件
  • · 安装 ComfyUI Manager 后，点击 Manager → Install Missing Custom Nodes
  • · 或手动去 GitHub 下载对应插件到 custom_nodes/
  • · 安装后记得重启 ComfyUI
模型加载失败 / 下拉列表为空？ ▼
  • · 确认模型放在了正确的目录下
  • · 确认模型文件完整（未损坏/未下完）
  • · 点击节点上的刷新按钮重新扫描
  • · 检查文件扩展名是 .safetensors 或 .ckpt
  • · 路径中不要有中文
生成的图片质量很差？ ▼
  • · 增加采样步数（Steps）到 25-30
  • · 调整 CFG 到 6-9 之间
  • · 添加质量词：masterpiece, best quality, highly detailed
  • · 添加负面提示词：worst quality, low quality, blurry
  • · 确保图片尺寸匹配模型（SD1.5 用 512，SDXL 用 1024）
  • · 尝试换一个采样器（推荐 dpmpp_2m_sde）
  • · 换一个更好的 Checkpoint 模型
如何导入别人分享的工作流？ ▼
  • · JSON 文件：直接拖拽到 ComfyUI 画布上
  • · PNG 图片：如果图片包含工作流元数据，拖拽到画布即可自动加载
  • · 菜单 → Load → 选择 JSON 文件
  • · 缺少的节点会显示为红色，按上方说明安装即可

  • · Windows: 使用 DirectML 版本的 PyTorch
  • · Linux: 安装 ROCm 驱动后使用原版
  • · 性能不如 NVIDIA，但基本功能都可用
  • · 启动参数：python main.py --directml

  • · 整合包用户：使用启动器的"更新"功能

📌 案例三：高清放大 — 将 512px 小图放大到 2K/4K

🎯 目标

生成小尺寸图片后进行 2-4 倍放大，获得清晰细腻的大图。

🔧 方案选择

节点链路：

KSampler → Latent Upscale (nearest-exact, 2x) → KSampler (denoise=0.5) → VAE Decode → Save

优点：显存占用低，速度快；缺点：细节略少

节点链路：

VAE Decode → Upscale Image (Using Model) → Save
额外需要：Load Upscale Model（推荐 4x-UltraSharp 或 RealESRGAN_x4plus）

优点：细节丰富，质量极高；缺点：显存占用较大

节点链路：

VAE Decode → Upscale Image (4x) → Scale Image (缩回 2x) → VAE Encode → KSampler (denoise=0.3-0.45) → VAE Decode → Save

先用超分模型放大获得细节，再缩回目标尺寸，最后用 KSampler 低去噪进行精修。这是目前效果最好的高清放大方案。

⚙️ 关键参数

第二次去噪强度0.3 - 0.55（太高会改变构图）
放大倍数2x 最稳定，4x 需要更多显存
超分模型真实感: RealESRGAN_x4plus
动漫: RealESRGAN_x4plus_anime_6B
通用: 4x-UltraSharp

📌 案例十四：视频转动画重绘 (Vid2Vid)

🎯 目标

🔗 工作流程

1. VHS_LoadVideo → 加载原始视频
2. ControlNet (LineArt/OpenPose) → 提取每帧结构
3. IPAdapter → 注入目标风格
4. AnimateDiff → 保证帧间连贯
5. KSampler → 逐帧/批量重绘

  • 帧间一致性：AnimateDiff 的 context 设置很关键，推荐 uniform 模式
  • 显存管理：长视频分段处理，每段 16-32 帧

🔬 生图原理图解

理解 Stable Diffusion 是如何一步步生成图片的

💡 潜空间是什么？

潜空间（Latent Space）是图片的压缩数学表示。一张 512×512 的图片在潜空间中只需要 64×64 的空间，运算量减少了 64 倍，这就是为什么 Stable Diffusion 能在消费级显卡上运行。

💡 去噪是怎么回事？

模型在训练时学会了"给噪声图去噪"的能力。生成时，从一张纯噪声图开始，模型根据文本条件逐步去除噪声，每一步让图像更清晰、更符合提示词，经过 20-30 步后就得到了最终图像。
